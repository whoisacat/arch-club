# Объект-значение (Value Object)

## Особенности

1. Value Object небольшие и легко создаются, их можно передавать в виде значения, 
   а не в виде ссылки на объект
2. Почти всегда имутабельные
3. Хранить лучше как внедренные (Embedded Value) значения или крупные сериализованные 
   (Serialized LOB) объекты.
4. Если в системе выполняется множество процедур сериализации в двоичный формат,
   следует подумать об оптимизации этого процесса для объектов-значений
